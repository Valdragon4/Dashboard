# Story 1.8: Interface de Gestion des Connexions Bancaires

## Status

Done

## Story

**As a** utilisateur,
**I want** une interface pour gérer mes connexions bancaires (ajouter, configurer, supprimer),
**so that** je peux contrôler quels comptes sont synchronisés automatiquement.

## Acceptance Criteria

1. Page `/bank-connections/` créée avec liste des connexions existantes affichant :
   - Nom du compte associé
   - Provider (Trade Republic, BoursoBank, Hello Bank)
   - Statut de synchronisation (pending, syncing, success, error)
   - Date/heure de la dernière synchronisation réussie
   - Nombre de transactions synchronisées (depuis le dernier SyncLog réussi)
   - Boutons d'action (synchroniser manuellement, modifier, supprimer)
2. Formulaire d'ajout de nouvelle connexion bancaire avec :
   - Sélection du provider (dropdown)
   - Sélection du compte associé (dropdown des comptes de l'utilisateur)
   - Nom de la connexion (optionnel, défaut: nom du compte)
   - Bouton "Suivant" pour passer à la configuration des credentials
3. Formulaire de configuration des credentials avec :
   - Champs spécifiques selon le provider sélectionné :
     - Trade Republic : `phone_number`, `pin`, `2fa_code` (optionnel)
     - BoursoBank : `username`, `password`, `2fa_code` (optionnel)
     - Hello Bank : `username`, `password`, `2fa_code` (optionnel)
   - Chiffrement côté serveur via `EncryptionService` avant sauvegarde
   - Gestion de l'authentification 2FA en deux étapes pour Trade Republic (si code non fourni)
   - Validation des credentials (test de connexion optionnel)
4. Affichage du statut de synchronisation pour chaque connexion :
   - Badge/indicateur visuel avec code couleur (vert=success, rouge=error, jaune=syncing, gris=pending)
   - Date/heure de la dernière synchronisation réussie (format lisible)
   - Nombre de transactions synchronisées depuis le dernier SyncLog réussi
   - Lien vers les logs de synchronisation (Story 1.10)
5. Bouton de synchronisation manuelle pour chaque connexion :
   - Appelle la tâche Celery `sync_bank_account()` de manière asynchrone
   - Affiche un message de confirmation immédiat
   - Met à jour le statut en temps réel (via polling ou WebSocket si disponible)
   - Gère l'authentification 2FA si nécessaire (affichage du formulaire 2FA)
6. Bouton de suppression de connexion avec confirmation :
   - Modal de confirmation avant suppression
   - Suppression de la `BankConnection` et des `SyncLog` associés
   - Mise à jour des comptes associés (retirer `bank_connection`)
   - Message de succès après suppression
7. Gestion de l'authentification 2FA dans l'interface (pour Trade Republic) :
   - Si `requires_2fa=True` retourné par `SyncService`, afficher un formulaire 2FA
   - Permettre la saisie du code 2FA et la re-synchronisation
   - Option pour renvoyer le code 2FA par SMS
8. Messages de succès/erreur pour les actions utilisateur :
   - Utilisation de Django messages (`messages.success()`, `messages.error()`)
   - Messages clairs et informatifs pour chaque action
   - Affichage des erreurs de synchronisation de manière lisible

- IV2: Les comptes sans connexion bancaire continuent d'être affichés normalement

**Formulaire BankConnectionForm** :

```python
from django import forms
from finance.models import BankConnection, Account
from finance.services.encryption_service import EncryptionService

class BankConnectionForm(forms.ModelForm):
    # Champs communs
    provider = forms.ChoiceField(choices=BankConnection.Provider.choices)
    account = forms.ModelChoiceField(queryset=Account.objects.none())
    account_name = forms.CharField(max_length=120, required=False)
    
    # Champs conditionnels selon le provider
    phone_number = forms.CharField(max_length=20, required=False)
    pin = forms.CharField(max_length=10, required=False, widget=forms.PasswordInput)
    username = forms.CharField(max_length=120, required=False)
    password = forms.CharField(max_length=255, required=False, widget=forms.PasswordInput)
    two_fa_code = forms.CharField(max_length=10, required=False, help_text="Code 2FA (optionnel)")
    
    class Meta:
        model = BankConnection
        fields = ["provider", "account_name", "auto_sync_enabled"]
    
    def __init__(self, user, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.user = user
        self.fields["account"].queryset = Account.objects.filter(owner=user)
    
    def clean(self):
        cleaned_data = super().clean()
        provider = cleaned_data.get("provider")
        
        # Valider les champs requis selon le provider
        if provider == BankConnection.Provider.TRADE_REPUBLIC:
            if not cleaned_data.get("phone_number") or not cleaned_data.get("pin"):
                raise forms.ValidationError("phone_number et pin sont requis pour Trade Republic")
        elif provider in [BankConnection.Provider.BOURSORAMA, BankConnection.Provider.HELLOBANK]:
            if not cleaned_data.get("username") or not cleaned_data.get("password"):
                raise forms.ValidationError("username et password sont requis")
        
        return cleaned_data
    
    def save(self, commit=True):
        instance = super().save(commit=False)
        instance.owner = self.user
        
        # Construire le dictionnaire de credentials selon le provider
        provider = self.cleaned_data["provider"]
        credentials = {}
        
        if provider == BankConnection.Provider.TRADE_REPUBLIC:
            credentials = {
                "phone_number": self.cleaned_data["phone_number"],
                "pin": self.cleaned_data["pin"],
            }
            if self.cleaned_data.get("two_fa_code"):
                credentials["2fa_code"] = self.cleaned_data["two_fa_code"]
        elif provider in [BankConnection.Provider.BOURSORAMA, BankConnection.Provider.HELLOBANK]:
            credentials = {
                "username": self.cleaned_data["username"],
                "password": self.cleaned_data["password"],
            }
            if self.cleaned_data.get("two_fa_code"):
                credentials["2fa_code"] = self.cleaned_data["two_fa_code"]
        
        # Chiffrer les credentials avant sauvegarde
        instance.encrypted_credentials = EncryptionService.encrypt_credentials(credentials)
        
        if commit:
            instance.save()
        
        return instance
```

**Vue bank_connections_list** :

```python
@login_required
def bank_connections_list(request: HttpRequest) -> HttpResponse:
    """Affiche la liste des connexions bancaires de l'utilisateur."""
    connections = BankConnection.objects.filter(owner=request.user).select_related("owner")
    
    # Récupérer les derniers SyncLog pour chaque connexion
    for connection in connections:
        last_success_log = SyncLog.objects.filter(
            bank_connection=connection,
            status=SyncLog.Status.SUCCESS
        ).order_by("-completed_at").first()
        
        connection.last_success_log = last_success_log
        connection.transactions_count = last_success_log.transactions_count if last_success_log else 0
    
    return render(request, "finance/bank_connections.html", {
        "connections": connections,
    })
```

```python
@login_required
def bank_connection_sync(request: HttpRequest, connection_id: int) -> HttpResponse:
    """Synchronise manuellement une connexion bancaire."""
    try:
        connection = BankConnection.objects.get(id=connection_id, owner=request.user)
    except BankConnection.DoesNotExist:
        messages.error(request, "Connexion bancaire introuvable.")
        return redirect("bank_connections_list")
    
    # Trouver le compte associé
    account = Account.objects.filter(bank_connection=connection).first()
    if not account:
        messages.error(request, "Aucun compte associé à cette connexion.")
        return redirect("bank_connections_list")
    
    # Appeler la tâche Celery de manière asynchrone
    from finance.tasks import sync_bank_account
    task_result = sync_bank_account.delay(account.id, sync_type=SyncLog.SyncType.MANUAL)
    
    # Vérifier si l'authentification 2FA est requise
    # Note: Pour une vraie implémentation, il faudrait attendre le résultat de la tâche
    # Pour l'instant, on retourne un message de confirmation
    
    messages.success(request, f"Synchronisation du compte {account.name} démarrée.")
    return redirect("bank_connections_list")
```
